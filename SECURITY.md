# Security Policy

## Supported Versions

Security updates are currently provided for:

| Version | Supported |
| ------- | --------- |
| 0.2.x   | Yes       |
| 0.1.x   | No        |
| < 0.1   | No        |

## Reporting a Vulnerability

Please use private reporting and do not open public issues for security problems.

- Preferred: GitHub private vulnerability report  
  `https://github.com/Orbis-Development/zora-lang/security/advisories/new`
- Alternative: email security contact if one is listed on the project website.

When reporting, include:

- A clear description of the issue
- Steps to reproduce
- A proof-of-concept (if available)
- Impact assessment
- Suggested fix (optional)

## Response Timeline

- Initial acknowledgment: within 72 hours
- Triage decision: within 7 days
- Fix and release timing: depends on severity and impact

## Disclosure

After a fix is released, coordinated disclosure details may be published in release notes.
